# Platform behavior

## Shared semantics

Linux and Windows use the same Python projection, status, backup, manifest,
frontmatter, and Codex TOML logic. Text generated by the CLI is UTF-8 with LF
newlines. User-facing relative paths always use `/`.

## Linux and Unix

Canonical shared surfaces use symlinks where supported. Managed directories
reject unexpected symlinks before mutation. The shell installer locates Python
3.11+, creates an isolated virtual environment, and installs the checkout in
editable mode.

## Windows

Canonical shared directories prefer Junctions. When Junction creation is not
available, the CLI uses an ownership-safe copy fallback with source identity and
content fingerprints. Reparse roots, unexpected targets, chained links, broken
links, and paths escaping a managed root are rejected.

Existing paths are compared with filesystem identity so a long path and its 8.3
alias are treated as the same object. Namespace prefixes and case differences
are normalized only when filesystem identity is unavailable.

PowerShell 5.1 and PowerShell 7 are both supported. CI forces UTF-8 Python mode
so subprocess output is decoded consistently.

## Codex TOML

`[projects.*]` tables are machine-local and preserved during apply. Status
filters them from the live file and normalizes CRLF/LF before comparing the
remaining general configuration.
